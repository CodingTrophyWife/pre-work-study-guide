HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Meta goes before head.
  • Link goes in head.
  • h1: Most important, header.
  • src: where to get a picture from, like a website.
  • img: picture, doesnt need ending tag.
  • alt: what do you want it to say if the picture doesnt load for some reason.
  • ul: bullet point lists.
  • ol: numbered lists.
  • li: list item.
  • figcaption: writing under the picture, put in
  • em: turns a word slanted. (cats LOVE lasagna)
  • strong: turns a word bold. (Cats HATE other cats)
  • p: if you want to write a paragraph or a couple sentences.
  • nested: two spaces further to the right of the element it is nested in.
  • a: if you want something clickable.
  • target: makes your link open with a new tab.
  • section: breaks up sections on your page.
  • figure: put an opening tag before the image you want to caption and a closing tag after it.
  • attribute: provide additional information about an element. "always use quotation marks around attributes.
  • element: is defined by a start tag, some content, and an end tag.
  • href: link a url to an image.

CSS

  • * Selector means that will be applied to all elements visible on the page
  • DO NOT REPEAT YOURSELF (DRY) is a coding princple that encourges developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained. DRY can greatly reduce the overall size and complexity of a codebase.

Git

  • git status: checks what branch we are currently on
  • git checklist -b branch name: creates a new branch and switches to it
  • Repository ("repo"): Every project on GitHub is called a respository. A folder for your project that contains all the files and their revision history.
  • Branch: Workspace where you can make changes that dont affect the live site. Create a new branch for your work, and name it to best describe the work (Example:"Content-3-26")
  • Markdown(.md): The way content files are formatted in GitHub. A way to write for the web using plain text; easily converts to HTML. Markdown files use .md file extention.
  • Commit changes: Saving a new version of a file you're working on. Whenever you add, edit, or delete a file, you're making a commit. Adds changes to your branch.
  • Pull Request ("PR"): Asking for changes on a branch to merge into another branch. Way to discuss and review your work with others.
  • Merge: When a PR is approved, changes are "Pulled in" or merged into main branch. Changes publish/deploy to the live site once this hasppens.
  • Issue: Way to report new tasks or content fixes. Track progress on project board.
  • GitHub will make you cry and want to rip your hair out!!!

JavaScript

  • All strings are surrounded by quotation marks
  • console.log: console refers to a test environment that developers use to check out their code. .log(), is a method, which means a set of instructions that can be executed by a computer. The .log() method will output whatever we add inside the parentheses to the console.
  • if statement is a way of introducing decision making into our code. The if statement, code will only execute if the statement meets a condition, or is true.
  • ===: Checks to see if two values are equal, are returns a boolean result true if the valuses are equal and false if the values are not equal.
  • Array: is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.
  • for loop: uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • A for loop contains the following three important statements:
    1. The first statement determines the starting point for our loop. Since we want to start at the beginning of the array, we declare a variable X and give it a value of 0 (because the first index of an array is always 0), as shown in the following line of code: var X = 0;
    2. The second statement is the condition. As long as the number in the variable X is less than the length of the array, the loop will keep running. This will allow the loop to execute one time for each item in the array. We use the length built-in array property on our shapes array to express that condition, as shown in the following line of code: X < shapes.length;
    3. The final statement is what allows the array to iterate over each item. Since indices follow a predictable pattern, we know the number increases by 1 each time. The code X++ is a shortcut way of writing X+1. Now everytime the code loops, X will increase by 1, as shown in the following line of code: X++
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
  • To declare a function, we use the function keyword and then give our function a unique name followed by parentheses. Then we add curly brackets {} that contain the code for the task we want to perform.
